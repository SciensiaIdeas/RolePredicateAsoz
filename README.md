<h2>Руководство оператора</h2>

1. Ваша программа записывает JSON-response от ASOZ API в текстовый файл ".asozres" (рекомендуется также скрытый атрибут). Файл находится в той же папке, что _RolePredicateAsoz.exe_
2. Ваша программа запускает процесс _RolePredicateAsoz.exe_ с единственным параметром — логическое выражение, содержащее допустимые слова в действующих ролях пользователя, а также логические операторы: 
- & (И)
- | (ИЛИ)
- ~ (НЕ)
- скобки ()

3. Содержащиеся слова в действующих ролях заменяются на true/false и подставляются в заданное выражение.
4. Результат процесса отражается в errorCode и может быть следующим: 

- 1 — если действующие роли соответствуют логическому выражению;
- 0 — если действующие роли не соответствуют логическому выражению;
- -1 — если возникла ошибка Asoz API и данные не были получены;
- -2 — если не существует файла ".asozres" или он имеет некорректную структуру JSON;
- -3 — если логическое выражение составлено неверно (ошибка в синтаксисе).

<h2>Пример</h2>

Пример файла ".asozres" представлен в релизе. Будем проверять несколько паттернов:

`F:\Documents\RolePredicateAsoz>RolePredicateAsoz.exe "ДУ-2 & ДУ-46 & АСУ СТ"`<br>
`F:\Documents\RolePredicateAsoz>echo %errorlevel%`<br>
`1`

`F:\Documents\RolePredicateAsoz>RolePredicateAsoz.exe "~ДУ-2 | ДУ-46 | ~АСУ СТ"`<br>
`F:\Documents\RolePredicateAsoz>echo %errorlevel%`<br>
`1`

`F:\Documents\RolePredicateAsoz>RolePredicateAsoz.exe "(ДУ-2 | ~ДУ-46) & ~АСУ СТ"`<br>
`F:\Documents\RolePredicateAsoz>echo %errorlevel%`<br>
`0`
